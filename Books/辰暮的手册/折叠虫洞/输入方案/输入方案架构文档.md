# 输入方案架构文档

## 概述

本文档描述 Folded Wormhole 项目的输入系统架构，基于 Unity New Input System 构建，提供统一的跨平台输入解决方案。

## 目录结构

```
UnityProject\Assets\TEngine\Runtime\Module\InputModule\
├── InputModule.cs              # 输入模块实现
├── IInputModule.cs              # 输入模块接口
├── InputSystem_Actions.cs       # 自动生成的输入代码
└── InputSystem_Actions.inputactions  # 输入动作资产
```

## 核心组件

### 1. 输入模块接口 (IInputModule)

```csharp
public interface IInputModule
{
    InputSystem_Actions.PlayerActions Player { get; }
    InputSystem_Actions.UIActions UI { get; }
}
```

### 2. 输入模块实现 (InputModule)

- 优先级：`8`
- 管理 Player 和 UI 两组输入动作
- 在 `OnInit()` 时启用输入，在 `Shutdown()` 时禁用

### 3. 访问入口

在 `GameModule.cs` 第 91 行添加了静态属性：

```csharp
public static IInputModule Input => _input ??= Get<IInputModule>();
```

通过 `GameModule.Input` 可以访问输入系统。

## 使用方式

### 1. 注册输入回调

```csharp
public class PlayerController : MonoBehaviour, IInputModule.IPlayerActions
{
    private void Start()
    {
        GameModule.Input.Player.AddCallbacks(this);
    }

    private void OnDestroy()
    {
        GameModule.Input.Player.RemoveCallbacks(this);
    }

    public void OnMove(InputAction.CallbackContext context)
    {
        Vector2 input = context.ReadValue<Vector2>();
    }

    public void OnAttack(InputAction.CallbackContext context)
    {
        if (context.performed)
        {
            // 攻击逻辑
        }
    }

    // 实现其他回调方法...
}
```

### 2. 动作状态说明

每个 InputAction 回调会触发三次：
- `started` - 按下瞬间
- `performed` - 动作完成
- `canceled` - 取消/释放

通常使用 `context.performed` 判断执行时机。

### 3. 启用/禁用输入

```csharp
// 禁用所有输入
GameModule.Input.Player.Disable();
GameModule.Input.UI.Disable();

// 重新启用
GameModule.Input.Player.Enable();
GameModule.Input.UI.Enable();
```

## 六边形移动特殊处理

项目使用六边形网格系统，专门的输入动作：

- **HexMove**: 用于六边形网格上的角色移动
- **HexSkillDir**: 用于技能释放方向输入
- **Skill1Press/Skill2Press**: 技能按键（对应扳机键）

## 热更说明

输入模块位于 `TEngine.Runtime` 程序集，属于非热更部分。业务逻辑中的输入处理在热更层通过 `GameModule.Input` 接口访问。
